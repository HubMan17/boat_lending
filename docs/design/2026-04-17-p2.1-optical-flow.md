# P2.1 Optical Flow Module

Date: 2026-04-17
Status: approved

## Goal

`companion/optical_flow.py` — sparse optical flow (Lucas-Kanade) on downward camera frames,
packed into MAVLink `OPTICAL_FLOW` messages for ArduCopter EKF velocity estimation in GPS-denied mode.

## Architecture

### Class `OpticalFlow`

```
OpticalFlow(width=640, height=480, fov_rad=0.7854)
  .process(frame: np.ndarray, ground_distance: float) -> FlowResult | None
```

- Stores `prev_gray` internally
- First call: saves frame, returns None
- Subsequent calls:
  1. `goodFeaturesToTrack` on prev (maxCorners=100, qualityLevel=0.3, minDistance=7)
  2. `calcOpticalFlowPyrLK(prev, curr, points)` → tracked points
  3. Median displacement dx, dy in pixels
  4. Convert to angular: `flow_x = dx/fx`, `flow_y = dy/fy` (radians, integrated)
  5. Convert to rate: `flow_rate_x = dx/fx/dt`, `flow_rate_y = dy/fy/dt` (rad/s)
  6. Quality = tracked/total * 255 (uint8)
  7. Redetect features when tracked count < 30
  8. Update prev_gray = curr

### Dataclass `FlowResult`

- `flow_x: float` — integrated angular displacement X (rad)
- `flow_y: float` — integrated angular displacement Y (rad)
- `flow_rate_x: float` — angular rate X (rad/s)
- `flow_rate_y: float` — angular rate Y (rad/s)
- `quality: int` — 0..255
- `ground_distance: float` — passthrough from caller (m)

### MavlinkSender additions

- `send_optical_flow(result: FlowResult)` — packs into `optical_flow_send()`
- `recv_altitude() -> float | None` — non-blocking `recv_match('GLOBAL_POSITION_INT')`,
  returns `relative_alt / 1000.0` (m). Source: barometer via SITL. In production: rangefinder.

### Companion loop integration (stage >= 2)

```python
frame = cam.read_frame()
alt = mav.recv_altitude()
if alt is not None:
    ground_distance = alt

flow = optical_flow.process(frame, ground_distance)
if flow:
    mav.send_optical_flow(flow)

detections = detector.detect(frame)
# ... rest of pipeline
```

Sequential in same loop — LK sparse <5 ms + ArUco <3 ms, well within 50 ms budget.

## Pixel-to-angular conversion

```
fx = fy = width / (2 * tan(fov_rad / 2))
flow_x = median_dx / fx          # radians (integrated over dt)
flow_y = median_dy / fy
flow_rate_x = flow_x / dt        # rad/s
flow_rate_y = flow_y / dt
```

Same intrinsics as ArucoDetector (fx=fy~772.5 for 640x480 FOV=0.7854).

## Smoke test

`scripts/smoke_optical_flow.py`:
- Generate two synthetic 640x480 frames with known pixel shift (e.g. 5px right, 3px down)
- Create OpticalFlow, call process twice
- Assert: flow direction matches shift sign, quality > 0, rate magnitude reasonable
- Test: first-frame returns None
- Test: zero-shift frame returns near-zero flow
